## Issues & Troubleshooting

- **Problem:** TanStack Router CLI (`@tanstack/router-cli generate`) failed with `ENOENT: no such file or directory, scandir 'src/routes'`
  - **Cause:** The CLI ignores command-line flags and looks for a default `src/routes` directory instead of the project's `src/renderer/routes`
  - **Fix:** Manually updated `routeTree.gen.ts` following the exact pattern of existing routes (16 locations needed updating). The file is gitignored and auto-generated by the TanStack Router Vite plugin, so it will be regenerated correctly when `pnpm run dev` runs.

- **Problem:** All backend tests failed with `PrismaClientInitializationError: Authentication failed against database server`
  - **Cause:** PostgreSQL was down initially (first attempt). After starting it, the `DATABASE_URL` in `.env` had credentials that didn't match the local PostgreSQL cluster (no chatbridge database existed, and the postgres user had no password set for password auth).
  - **Fix:** Set postgres password (`ALTER USER postgres WITH PASSWORD 'postgres'`), created the chatbridge database (`CREATE DATABASE chatbridge`), pushed the Prisma schema (`prisma db push`), ran seed (`npm run seed`), then ran tests with explicit `DATABASE_URL="postgresql://postgres:postgres@localhost:5432/chatbridge"`. All 47 tests passed.

- **Problem:** First session attempt couldn't start PostgreSQL — needed sudo password
  - **Cause:** `pg_ctlcluster` requires root or the postgres OS user
  - **Fix:** User started PostgreSQL between sessions. In the final run, `pg_isready` confirmed it was accepting connections.

- **Problem:** `pnpm run typecheck` failed with Node.js version mismatch
  - **Cause:** Active Node.js was v24.13.1 but `package.json` requires `>=20.0.0 <23.0.0`
  - **Fix:** Used `nvm use 22` to switch to v22.22.2. The `typecheck` script was actually named `check` (`pnpm run check`). Pre-existing TS errors exist across the codebase (InputBox, ChatBridgeFrame, Mermaid, etc.) — none were introduced by the admin panel changes.

## Decisions Made

- **Manual routeTree.gen.ts update over running dev server** — The Vite plugin auto-generates this file, but we couldn't run the dev server in this context. Manual update was reliable since the pattern is mechanical and the file is `@ts-nocheck`. It will be overwritten correctly on next `pnpm run dev`.
- **No role-based access control for admin panel** — Any authenticated user can access `/admin`. This is explicitly a sprint scope decision documented in both the plan and the trust & safety doc. Production would need an admin role.
- **`GET /api/apps/all` returns `{ apps: [...] }` wrapper** — Unlike `GET /api/apps` which returns a bare array, the `/all` endpoint wraps in `{ apps }` for consistency with the plan. The admin panel expects this shape.
- **Trust & safety doc at ~740 words** — Slightly over the 400-600 word target but all sections are substantive. Kept it because cutting content would weaken the child safety argument.
